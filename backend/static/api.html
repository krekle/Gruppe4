SheepFarmer 3000

This is the API documentation for the SheepFarmer 3000 API!

The base url for all the requests is  http://dev.krekle.net:8081/

  • This url will return this page
  • (As you already know)

API-endPoints

 

'/status'
Returns the server status.
No parameters required.

 

'/login'
HttpGet will return a html-form for HttpPost request and requires no parameters
HttpPost will return a session token specific for a user on successful authentification
this requires:
Email, as 'email' and Password, as 'pswd' or
Session token, as 'token'

 

'/register'
HttpGet will return a html-form for HttpPost request and requires no parameters
HttpPost will return a session token specific for a user on successful user creation.
Post requires parameters:
Email, as 'email'
Password, as 'pswd'
Name, as 'name'
Telephone, as 'phone'
Address, as 'address'
Vara, as 'vara'

 

'/notification
This is a HttpGet endpoint for getting notifications
If only the token parameter is sent with the request, it will return a JsonArray with notifications for a user.
If token and sheepid is sent with the request, it will return a  JsonArray with notifications for sheep with sheepid.

 

'/sheep'
This is a HttpGet endpoint for getting sheeps
If only the token parameter is sent with the request, it will return a JsonArray with all sheeps for a user.
If token and sheepid is sent with the request, it will return a  single sheep with sheepid.

 

'/sheep/add'
This is a HttpGet endpoint for adding new sheep
This requires token for user validation and data for a sheep;
'name', 'age', 'gender', 'weight' the rest is added by the server.
